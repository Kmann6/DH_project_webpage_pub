    • Information Recording and Data Storage

    The author is committed to promoting sustainability in all aspects of their work and operations. All files are saved in multiple locations, including the public repository available on GiHub. Additioanllly, the README file discusses the research methods and information recording standards. All files used for the creation of this website was done using open source products that can be replicated using the descriptions in README. This site uses jpeg files for all images and csv files for datasets. This site is intended for broad use; this is a crowd sourcing page for the memories individuals have about learning history from popular culture consumer products. However, the datasets will be available for students and researchers interested in popular culture, historical memory, and history education, and other aplications within digital humanities. The repository and its dependacies are open source annd files can be accessed via HTML and csv. All files are held in GitHub and available for acces via a static URL which will remain active without constant maintainence. Maintainence and updates will be completed as available. The author used Jupyter Notebook to clean datasets and the processes are documented in the Jupyter Notebook and README file for replication. File storage is redundant(local, cloud, and Git repository) and versioning is available on GitHub.

    Environmental Impact

    The author is committed to minimizing the environmental impact of their work and operations. The website is hosted on GitHub Pages, which uses renewable energy sources to power its servers. Additionally, the website is designed to be lightweight and optimized for fast loading times, which reduces the amount of energy required to access it. The author also encourages users to access the website using energy-efficient devices and to minimize their own environmental impact while browsing.

    Accessibility

    The author is committed to ensuring that the website is accessible to all users. The website is designed to be compatible with screen readers and other assistive technologies. Additionally, the website follows web accessibility guidelines to ensure that all users can navigate and interact with the content effectively. If issues arise, please let the author know using the contact information provided.
